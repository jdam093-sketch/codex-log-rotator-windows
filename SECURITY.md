# Security Policy

## Reporting a vulnerability

Use GitHub private vulnerability reporting through the repository's **Security** tab when it is available.

Do not include real Codex logs, SQLite databases, manifests, backup archives, prompts, local paths, project names, credentials, tokens, or personal information in a public issue. If private reporting is unavailable, open a public issue that contains only a request for a private contact channel and no vulnerability details or sensitive data.

## Scope

Security reports should concern the behavior of this repository's script, tests, or release artifacts. This unofficial project cannot accept reports about Codex itself; report those through OpenAI's official security channels.
